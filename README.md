# 🎓 PROG2 - Computer Programming 2

**Enrichment Course Interactive Presentations & Debugging Sessions**

---

## 📚 Course Overview

PROG2 is a comprehensive Computer Programming 2 course featuring interactive slide presentations, exercises, exams, and hands-on debugging sessions. This repository contains all course materials designed for students at the **University of San Jose-Recoletos**.

### Course Level
- **Type:** Enrichment Course for Computer Programming
- **Duration:** 8 weeks
- **Format:** Interactive HTML presentations with Reveal.js

---

## 🚀 Quick Start - Access the Course

### **For Students:**
👉 **Open the course here:** https://tableaprogramming-rgb.github.io/lesson-ecprog/

This will take you to the **Master Navigation Index** where you can access all lessons:

1. Select any **Day (1-8)** from the grid
2. Choose your **Exercise, Exam, or Debug Session**
3. Use **Spacebar** or **Arrow Keys** to navigate slides
4. Press **F** for fullscreen mode
5. Press **?** to see all keyboard shortcuts

## 📖 Course Structure

### **Days 1-3: Fundamentals**
- **Day 1:** Code Tracing & Variable Concepts
  - 5 interactive exercises
  
- **Day 2:** Arrays & Data Structures
  - 10 comprehensive exercises
  
- **Day 3:** Advanced Concepts & Assessment
  - 4 exam questions with solutions
  - 5 practice exercises

### **Days 4-6: Intermediate Topics**
- **Day 4:** Advanced Debugging Techniques
- **Day 5:** Functions in C (Deep Dive)
- **Day 6:** Hands-On Function Exercises

### **Days 7-8: Advanced & Practical**
- **Day 7:** 7 advanced exercises
- **Day 8:** 7 comprehensive exercises + **9 Debugging Sessions**

### **Day 8 Afternoon - Debugging Sessions** 🐛
Interactive debugging exercises where you:
1. Identify bugs in C code
2. Understand why errors occur
3. Trace execution step-by-step
4. Write and verify fixes

**Debugging Topics:**
- Debug 1: Off-by-One Errors in Loops
- Debug 2: Wrong Function Parameters
- Debug 3-8: Additional debugging scenarios
- Debug Bonus: Extra challenge

## 🎯 How to Use Different Material Types

### 📝 Exercises
1. Read the problem statement
2. Study the buggy or incomplete code
3. Work through the step-by-step guide
4. Complete the coding task
5. Check your work against the answer key

### 🧪 Exams
1. Review the problem context
2. Identify the bug or complete the task
3. Follow the 5-step task list:
   - Find the bug(s)
   - Explain why it fails
   - Trace execution
   - Write the fix
   - Verify the solution

### 🐛 Debugging Sessions
1. **Slide 1-2:** Problem Introduction & Context
2. **Slide 3:** Buggy Code Preview
3. **Slide 4:** Bug Identification
4. **Slide 5:** Task Instructions (5 steps)
5. **Slide 6:** Explanation of the Issue
6. **Slide 7:** Fixed Code with Solutions
7. **Slide 8:** Learning Summary
