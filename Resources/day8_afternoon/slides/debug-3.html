The Context

Goal:

Calculate the average grade for a specific student (row 2) in a 4×3 grade matrix

Row 2 grades: [78, 85, 82] — Expected average: 81.67

Expected Output:

Average: 81.67

The Buggy Code

What does this code actually produce?

#include <stdio.h>

int main() {
    int grades[4][3] = {
        {85, 90, 78},
        {92, 88, 95},
        {78, 85, 82},
        {88, 92, 89}
    };

    int row = 2;
    int sum;  // ❌ NOT INITIALIZED!

    for (int j = 0; j < 3; j++) {
        sum += grades[row][j];
    }

    double average = sum / 3.0;
    printf("Average: %.2f\n", average);
    return 0;
}

Actual Output

Unpredictable garbage value

Problem:

Expected: Average: 81.67

Got: Random different value each run

3 / 8

The Bug: Uninitialized Variable

What happens without initialization?

🐛 The Problem

int sum; declares but doesn't initialize
Variable contains garbage/random value from memory

💥 Why It Fails

sum += grades[row][j]
Adding to undefined value → unpredictable result

✓ The Fix

int sum = 0; initialize to zero

Why Variables Need Initialization

Understanding memory and garbage values

What Is Memory?

Memory contains leftover data from previous programs
Uninitialized variables read that leftover data

The Result

Each run produces different garbage value
Makes bugs hard to reproduce and debug

Best Practice

ALWAYS initialize variables when declaring

The Fixed Code

With proper initialization

#include <stdio.h>

int main() {
    int grades[4][3] = {
        {85, 90, 78},
        {92, 88, 95},
        {78, 85, 82},
        {88, 92, 89}
    };

    int row = 2;
    int sum = 0;  // ✓ INITIALIZED

    for (int j = 0; j < 3; j++) {
        sum += grades[row][j];
    }

    double average = sum / 3.0;
    printf("Average: %.2f\n", average);
    return 0;
}

Execution Trace

sum = 0 (initial)

j=0: sum = 78

j=1: sum = 163

j=2: sum = 245

Learning Summary

Key takeaways from this debugging exercise

ALWAYS Initialize Variables

Declare AND initialize in same statement

Accumulation Variables

Variables used in += must start at 0 or identity value

Flags Should Start False

int found = 0; not uninitialized

Hard to Debug

Different every run → enable compiler warnings!
