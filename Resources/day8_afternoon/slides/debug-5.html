The Buggy Code

What does this code actually produce?

#include <stdio.h>

int main() {
    int matrix[2][3];
    int num = 1;

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            matrix[j][i] = num;  // ❌ BUG: SWAPPED!
            num++;
        }
    }

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", matrix[i][j]);
        }
        printf("\n");
    }

    return 0;
}

The Bug: Swapped Indices

Using [j][i] instead of [i][j]

🐛 The Problem

Assignment uses: matrix[j][i]
Should use: matrix[i][j]
Result: Data written to wrong locations

💥 What Happens

Data gets written to transposed positions
Also causes out-of-bounds when j >= 2 (for [j][i])

✓ The Fix

matrix[i][j] = num;
Convention: [row][column]

Why Index Order Matters

Understanding [i][j] convention

The Convention

[i][j] means [row][column]
i is row number (0 to rows-1)
j is column number (0 to cols-1)

For [2][3] Matrix

Valid: [0][0] to [0][2], [1][0] to [1][2]
Invalid: [0][3], [1][3], [2][0], [2][1], [2][2]

Using [j][i]

Treats column as row and row as column
Causes out-of-bounds or wrong locations

Correct index usage

#include <stdio.h>

int main() {
    int matrix[2][3];
    int num = 1;

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            matrix[i][j] = num;  // ✓ FIXED
            num++;
        }
    }

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", matrix[i][j]);

Learning Summary

Key takeaways from this debugging exercise

Always Use [i][j]

i for rows, j for columns

Outer Loop = Rows

for (int i...) usually outer

Inner Loop = Columns

for (int j...) usually inner
