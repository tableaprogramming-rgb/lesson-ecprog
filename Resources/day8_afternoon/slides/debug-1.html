The Buggy Code

What does this code actually produce?

#include <stdio.h>

int main() {
    int matrix[3][3] = {
        {10, 20, 30},
        {40, 50, 60},
        {70, 80, 90}
    };

    int max = 0;
    for (int i = 0; i <= 3; i++) {
        for (int j = 0; j <= 3; j++) {
            if (matrix[i][j] > max) {
                max = matrix[i][j];
            }
        }
    }

    printf("Maximum value: %d\n", max);
    return 0;
}

Actual Output

Segmentation fault (or crash)

Problem:

Expected: Maximum value: 90

Got: Program crash or undefined behavior

The Bugs in This Code

Three issues to identify

🐛 Bug #1: Loop Condition

i <= 3 should be i < 3
Array indices are 0, 1, 2 (not 0-3)

🐛 Bug #2: Inner Loop Condition

j <= 3 should be j < 3
Accesses invalid index [i][3]

🐛 Bug #3: Initial Value

int max = 0; fails if all values are negative
Better: int max = matrix[0][0];

Why This Happens

Understanding off-by-one errors

Array Indexing Rule

Array of size N: Valid indices are 0 to N-1
3×3 array: Valid indices are 0-2 for both rows and columns
Using index 3: Accesses memory OUTSIDE the array

Loop Condition < vs <=

< 3: Iterates i = 0, 1, 2 ✓
<= 3: Iterates i = 0, 1, 2, 3 ✗ (index 3 is out of bounds)

Consequences

• Accessing invalid memory
• Undefined behavior (crash or garbage values)
• Hard to debug (works sometimes, not others)

The Fixed Code

Corrected version with proper bounds

#include <stdio.h>

int main() {
    int matrix[3][3] = {
        {10, 20, 30},
        {40, 50, 60},
        {70, 80, 90}
    };

    int max = matrix[0][0];  // FIX
    for (int i = 0; i < 3; i++) {  // FIX
        for (int j = 0; j < 3; j++) {  // FIX
            if (matrix[i][j] > max) {
                max = matrix[i][j];
            }
        }
    }

    printf("Maximum value: %d\n", max);
    return 0;
}

Key Changes

✓ Loop: i < 3

Valid range: 0, 1, 2

✓ Loop: j < 3

Valid range: 0, 1, 2

✓ Init: matrix[0][0]

Handles negative values

Key takeaways from this debugging exercise

Always Use < Not <=
