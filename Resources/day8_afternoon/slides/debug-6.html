The Context

Goal:

Calculate the average value of all elements in a 2×3 matrix, showing decimal precision

Matrix: [[10,20,30],[40,50,60]] — Sum = 210, Average = 35.0

Expected Output:

Average: 35.0

2 / 8

The Buggy Code

What does this code actually produce?

#include <stdio.h>

int main() {
    int matrix[2][3] = {
        {10, 20, 30},
        {40, 50, 60}
    };

    int sum = 0;
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            sum += matrix[i][j];
        }
    }

    int average = sum / 6;  // ❌ BUG: Integer division
    printf("Average: %d\n", average);
    return 0;
}

Actual Output

Average: 35 (no decimal!)

Problem:

Expected: Average: 35.0

Got: Average: 35 (decimal precision lost)

3 / 8

The Bug: Integer Division

Lost decimal precision

🐛 The Problem

int average = sum / 6;
Both operands are integers → integer division
210 / 6 = 35 (loses .0)

💥 What Happens

Decimal part is truncated (thrown away)
For 211 / 6: Should be 35.166... but int gives 35

✓ The Fix

double average = sum / 6.0;
Make one operand a double

Understanding Type Conversion

When and why it happens

Integer Division Rule

int / int = int
double / int = double
210 / 6 = 35 (not 35.0)

How to Get Floating Point

Make divisor a double: sum / 6.0
Cast numerator: (double)sum / 6
Use double variable: double avg = sum / 6.0

6 / 8

The Fixed Code

Using floating-point arithmetic

#include <stdio.h>

int main() {
    int matrix[2][3] = {
        {10, 20, 30},
        {40, 50, 60}
    };

    int sum = 0;
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            sum += matrix[i][j];
        }
    }

    double average = sum / 6.0;  // ✓ FIXED
    printf("Average: %.1f\n", average);
    return 0;
}
