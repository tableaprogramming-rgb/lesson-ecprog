The Context

Goal:

Search a 3×3 matrix for a target value (5) and report whether it was found or not

Matrix: [[1,2,3],[4,5,6],[7,8,9]] — value 5 is at position [1][1]

Expected Output:

Value 5 was found

The Buggy Code

What does this code actually produce?

#include <stdio.h>

int main() {
    int matrix[3][3] = {
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9}
    };

    int target = 5;
    int found = 0;

    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            if (matrix[i][j] == target) {
                found = 1;     // Found it!
            } else {
                found = 0;     // ❌ BUG: Reset!
            }
        }
    }

    if (found) {
        printf("Value %d was found\n", target);
    } else {
        printf("Value %d was not found\n", target);
    }

    return 0;
}

Actual Output

Value 5 was not found

Problem:

Expected: Value 5 was found

Got: Value 5 was not found (flag gets reset!)

3 / 8

The Bug: Logic Error

Else clause overwrites the found flag

🐛 The Problem

If value 5 is found, found = 1
But then hits other elements and found = 0
Final result: found remains 0 (incorrect!)

💥 Why It's Wrong

Else clause resets flag even after finding value
State gets lost over loop iterations

✓ The Fix

Remove the else clause: Don't reset once found
Or: Add break; to exit early

Flag Purpose

Common Pattern

The Fixed Code

Two solutions shown

// Solution A: Remove else
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (matrix[i][j]==target) {
            found = 1;
        }
        // No else clause!
    }
}

// Solution B: Add break
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (matrix[i][j]==target) {
            found = 1;
            break;  // Exit inner
        }
    }
    if (found) break;  // Exit outer
}

Output

✓ Both Give

Value 5 was found
