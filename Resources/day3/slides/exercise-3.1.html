Exercise 3.1: Debugging

Logic Error - Backwards Condition

Find the bug in the condition

The Context

Goal:

Check if a number is positive

Expected Behavior:

If x = 10: Print "x is positive"
If x = -5: Print "x is negative"

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int x = 10;

    if (x < 0) {
        printf("x is positive\n");
    } else {
        printf("x is negative\n");
    }

    return 0;
}

Console Output

x is negative

Problem:

x = 10 (positive), Expected: "x is positive"

Where is the bug?

The Bug

Line 6 has the error

#include <stdio.h>

int main() {
    int x = 10;

    if (x < 0) {
        printf("x is positive\n");
    } else {
        printf("x is negative\n");
    }

    return 0;
}

🐛 BUG: Backwards logic

What's Wrong:

Condition checks if x < 0
But prints "positive"

What Happens:

10 < 0? FALSE
Goes to else block
Prints "x is negative"
But 10 IS positive!

Result:

Condition logic is reversed
Wrong!

How do we fix this?

The Fix

Change < to >

#include <stdio.h>

int main() {
    int x = 10;

    if (x > 0) {
        printf("x is positive\n");
    } else {
        printf("x is negative\n");
    }

    return 0;
}

✓ FIXED: Changed < to >

What Changed:

< (less than) changed to
> (greater than)

Now It Means:

"If x is greater than 0
print positive"

Result:

10 > 0? TRUE
Prints "x is positive"
Correct!

Correct output →

Verification

Fixed code produces correct output

Buggy Code Output

x is negative  ✗ WRONG
↓

Fixed Code Output

x is positive  ✓ CORRECT

The fix works! Bug solved.

Key Lesson

Logic Errors:

Condition syntax is correct, but logic is backwards
Code runs without errors, but gives wrong results

Comparison Operators:
• < - Less than (smaller)
• > - Greater than (larger)
• <= - Less than or equal
• >= - Greater than or equal

How to Catch This:
Test your code! Run it with different values
Does it give the right answer?
