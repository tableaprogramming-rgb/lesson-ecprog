Exam Question 1.2

Loop Boundary with Increment

25 Points - Intermediate

The Context

Goal:

Calculate the factorial of 5 (written as 5!)

Calculation:

5! = 5 × 4 × 3 × 2 × 1 = 120

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int result = 1;

    for (int i = 1; i < 5; i++) {
        result = result * i;
    }

    printf("5! = %d\n", result);

    return 0;
}

Actual Output

5! = 24

Problem:

Expected 120, but got 24

Trace through the loop

Your Task (25 Points)

Complete the following:

  1. Find the bug: What's wrong with line 6?
  2. Trace it: Show what gets multiplied at each iteration (i=1, 2, 3, 4, 5)
  3. Explain why: Why does 24 appear instead of 120?
  4. Write the fix: Change only the loop condition (don't change the body)
  5. Verify: Show the corrected code produces 5! = 120

Remember: Show all your work on your exam answer sheet

Include your trace showing each iteration

Think About:

Question 1: What number is missing from the multiplication? (Hint: We calculated 1×2×3×4 but needed 1×2×3×4×?)

Question 2: When does the loop stop with i < 5? (i=1,2,3,4 then stop, or i=1,2,3,4,5 then stop?)

Question 3: What condition would make the loop include i=5?

Fix Hint: This is an off-by-one error. Change the boundary condition.
