Exam Question 2.1

Leap Year Logic Error

25 Points - Challenge

The Context

Goal: Determine if a year is a leap year

Leap Year Rules:
• Divisible by 400? → Leap year ✓
• Divisible by 100 (but not 400)? → NOT leap year ✗
• Divisible by 4 (but not 100)? → Leap year ✓
• Otherwise? → NOT leap year ✗

Test Cases:

YearShould Be
2000✓ Leap (div by 400)
1900✗ NOT leap (div by 100)
2020✓ Leap (div by 4)
2019✗ NOT leap

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int year = 1900;

    if (year % 4 == 0 ||
        year % 100 != 0 ||
        year % 400 == 0) {
        printf("Leap year\n");
    } else {
        printf("Not a leap\n");
    }
    return 0;
}

Test Results

2000: Leap year ✓
1900: Leap year ✗
2020: Leap year ✓
2019: Not a leap ✓

Problem:

Year 1900 should print "Not a leap" but prints "Leap year"

This one is tricky! Complex logic needed.

Your Task (25 Points)

Complete the following:

  1. Find the bug: What's wrong with the if condition?
  2. Analyze: Why does 1900 incorrectly print "Leap year"?
  3. Trace 1900: Test each part of the condition and show the OR result
  4. Write the fix: Rewrite the complete if condition correctly
  5. Verify: Test your fixed condition with all 4 cases above
