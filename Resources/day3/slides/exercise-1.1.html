Exercise 1.1: Debugging

Off-by-One Error in Loop

Find the bug, understand why, and fix it

The Context

Goal:

Calculate the sum of numbers from 1 to 5

Calculation:

1 + 2 + 3 + 4 + 5 = 15

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int sum = 0;

    for (int i = 1; i < 5; i++) {
        sum = sum + i;
    }

    printf("Sum: %d\n", sum);

    return 0;
}

Console Output

Sum: 10

Problem:

Expected 15, but got 10

Where is the bug?

The Bug

Line 6 has the error

#include <stdio.h>

int main() {
    int sum = 0;

    for (int i = 1; i < 5; i++) {
        sum = sum + i;
    }

    printf("Sum: %d\n", sum);

    return 0;
}

🐛 BUG: i < 5

What's Wrong:

The condition i < 5 means:
"Loop while i is LESS than 5"

What Happens:

Loop runs: i = 1, 2, 3, 4
But STOPS before i = 5

Result:

Sum = 1+2+3+4 = 10
Missing the 5!

How do we fix this?

The Fix

Change < to <=

#include <stdio.h>

int main() {
    int sum = 0;

    for (int i = 1; i <= 5; i++) {
        sum = sum + i;
    }

    printf("Sum: %d\n", sum);

    return 0;
}

✓ FIXED: i <= 5

What Changed:

< changed to <=

Now It Means:

"Loop while i is LESS THAN OR EQUAL TO 5"

Loop Now Runs:

i = 1, 2, 3, 4, 5
Includes the 5!

Correct output →

Verification

Fixed code produces correct output

Buggy Code Output

Sum: 10  ✗ WRONG
↓

Fixed Code Output

Sum: 15  ✓ CORRECT

The fix works! Bug solved.

Key Lesson

Off-by-One Errors:

Very common in loops. The boundary conditions (<, <=, >, >=) matter!

Remember:
• i < 5 means: stop BEFORE reaching 5
• i <= 5 means: INCLUDE 5

How to catch this:
1. Trace the code (write down each iteration)
2. Count: How many times should the loop run?
3. Compare: Does my loop count match?
