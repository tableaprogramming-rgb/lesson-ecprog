Exercise 2.1: Debugging

AND vs OR Operator

Find the bug with && and ||

The Context

Goal:

Check if x is in range: 0 < x < 10

Expected Behavior:

If x = 5: Print "In range"
If x = 15: Print "Out of range"

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int x = 15;

    if (x > 0 || x < 10) {
        printf("In range\n");
    } else {
        printf("Out of range\n");
    }

    return 0;
}

Console Output

In range

Problem:

x = 15, Expected: "Out of range"

Where is the bug?

The Bug

Line 6 has the error

#include <stdio.h>

int main() {
    int x = 15;

    if (x > 0 || x < 10) {
        printf("In range\n");
    } else {
        printf("Out of range\n");
    }

    return 0;
}

🐛 BUG: || should be &&

What's Wrong:

Used OR (||)
Should use AND (&&)

What Happens:

15 > 0? YES
So OR is TRUE!
(Only needs one TRUE)

Result:

Accepts 15 as in range
Wrong!

How do we fix this?

The Fix

Change || to &&

#include <stdio.h>

int main() {
    int x = 15;

    if (x > 0 && x < 10) {
        printf("In range\n");
    } else {
        printf("Out of range\n");
    }

    return 0;
}

✓ FIXED: && instead of ||

What Changed:

|| changed to &&

Now It Means:

BOTH conditions
must be TRUE

Result:

15 > 0 AND 15 < 10?
First true, 2nd false
AND = FALSE
Correct!

Correct output →

Verification

Fixed code produces correct output

Buggy Code Output

In range  ✗ WRONG
↓

Fixed Code Output

Out of range  ✓ CORRECT

The fix works! Bug solved.

Key Lesson

AND vs OR:

|| (OR) = At least ONE must be true
&& (AND) = BOTH must be true

For Range Checking:
• Use AND (&&) for "between" ranges
• Use OR (||) for "outside" ranges

Remember:
0 < x < 10 means x > 0 AND x < 10
