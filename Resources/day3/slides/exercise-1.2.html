Exercise 1.2: Debugging

Assignment vs Comparison

Find the bug with = and ==

The Context

Goal:

Check if x equals 5

Expected Behavior:

If x = 5: Print "x is 5"
If x = 3: Print "x is not 5"

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    int x = 3;

    if (x = 5) {
        printf("x is 5\n");
    } else {
        printf("x is not 5\n");
    }

    return 0;
}

Console Output

x is 5

Problem:

Expected: "x is not 5"

Where is the bug?

The Bug

Line 6 has the error

#include <stdio.h>

int main() {
    int x = 3;

    if (x = 5) {
        printf("x is 5\n");
    } else {
        printf("x is not 5\n");
    }

    return 0;
}

🐛 BUG: x = 5

What's Wrong:

= is ASSIGNMENT
Not comparison!

What Happens:

x = 5 sets x TO 5
Then it checks if 5 is true
(Any non-zero value is TRUE)

Result:

Always prints "x is 5"
Wrong!

How do we fix this?

The Fix

Change = to ==

#include <stdio.h>

int main() {
    int x = 3;

    if (x == 5) {
        printf("x is 5\n");
    } else {
        printf("x is not 5\n");
    }

    return 0;
}

✓ FIXED: x == 5

What Changed:

= changed to ==

Now It Means:

== is COMPARISON
Checks if x equals 5

Result:

x stays 3
3 == 5? FALSE
Prints correct answer!

Correct output →

Verification

Fixed code produces correct output

Buggy Code Output

x is 5  ✗ WRONG
↓

Fixed Code Output

x is not 5  ✓ CORRECT

The fix works! Bug solved.

Key Lesson

Assignment vs Comparison:

= is ASSIGNMENT (puts a value into a variable)
== is COMPARISON (checks if two values are equal)

Remember:
• In if statements, you almost always want ==
• Using = accidentally changes the variable!

Compiler Tip:
Some compilers warn about this. Enable warnings!
