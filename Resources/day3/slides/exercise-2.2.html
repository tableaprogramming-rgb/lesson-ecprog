Exercise 2.2: Debugging

Missing Loop Increment

Find the bug in the loop

The Context

Goal:

Print numbers from 1 to 5, one per line

Expected Behavior:

Output should be: 1
2
3
4
5

The Buggy Code

What does this code actually do?

#include <stdio.h>

int main() {
    for (int i = 1; i <= 5; ) {
        printf("%d\n", i);
    }

    return 0;
}

Console Output

1
1
1
1
1
... (infinite)

Problem:

Program prints 1 forever (infinite loop)

Where is the bug?

The Bug

Line 4 has the error

#include <stdio.h>

int main() {
    for (int i = 1; i <= 5; ) {
        printf("%d\n", i);
    }

    return 0;
}

🐛 BUG: Missing i++

What's Wrong:

The for loop has no increment
i is stuck at 1

What Happens:

i starts at 1
1 <= 5? YES
Print 1
No increment!
Loop again with i=1

Result:

Infinite loop printing 1
Wrong!

How do we fix this?

The Fix

Add i++ to the loop

#include <stdio.h>

int main() {
    for (int i = 1; i <= 5; i++) {
        printf("%d\n", i);
    }

    return 0;
}

✓ FIXED: Added i++

What Changed:

Added i++ in the increment part

Now It Works:

After each iteration,
i increases by 1

Loop Iterations:

i=1 (print 1)
i=2 (print 2)
... i=5 (print 5)
i=6 (exit loop)

The fix works! Bug solved.

Key Lesson

Loop Structure:

for (init; condition; increment)
All three parts are important!

Remember:
• init: Set starting value (e.g., i=1)
• condition: Keep looping while true (e.g., i<=5)
• increment: Change value each iteration (e.g., i++)

Without increment:
The loop variable never changes → Infinite loop!
