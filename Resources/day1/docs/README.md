# PROG2 Week 1: Code Reading Exercises

## What's In This Folder

This folder contains a complete set of exercises designed to teach students to **read code carefully** rather than rely on shortcuts like AI or internet searches.

---

## Files Overview

### 1. **WEEK1_EXERCISES.md** (Student-Facing)
The main exercise document for students.

**Contains:**
- Exercise Set 1: Code Tracing (5 progressively harder exercises)
- Exercise Set 2: Error Finding (4 buggy code snippets)
- Exercise Set 3: Output Prediction (5 exercises to check understanding)
- Exercise Set 4: Code Matching (bonus - matching code to outputs)

- If-else statements

## Quick Start for Instructors

### Preparation (30 minutes before class)
1. Read INSTRUCTOR_GUIDE.md (especially "Opening Speech")
2. Review WEEK1_ANSWER_KEY.md to know what to expect
3. Print WEEK1_EXERCISES.md (or share digitally)
4. Prepare whiteboard/screen for modeling Exercise 1.1

### During Class (50 minutes)
1. **Opening (5 min):** Read the opening speech from INSTRUCTOR_GUIDE.md
2. **Modeling (10 min):** Work through Exercise 1.1 together on whiteboard
3. **Guided Practice (20 min):** Students do Exercises 1.2-1.3 while you circulate
4. **Independent Work (10 min):** Students do Exercises 1.4-1.5
5. **Wrap-up (5 min):** Collect, discuss one common mistake

### After Class
- Review student work using WEEK1_ANSWER_KEY.md
- Note patterns of misunderstanding
- Identify 1-2 errors to address next class
- Decide whether to assign remaining exercises as homework

## What Students Need

**Materials:**
- Printed copy of WEEK1_EXERCISES.md (or digital access)
- Pen and paper (for traces and explanations)
- Dev C or similar C compiler (only after predictions are made)

**Mindset:**
- Willingness to slow down and think carefully
- Understanding that this isn't about speed, it's about understanding
- Acceptance that not knowing is okay - that's why they're here to learn

---

## Learning Outcomes

By completing Week 1 exercises, students will:

✅ **Understand:**
- How to read code systematically
- How variables change through execution
- How loops and conditionals work together

✅ **Be Able to:**
- Trace code step-by-step
- Predict program output without running it
- Identify and explain common code errors
- Show their reasoning clearly

✅ **Believe:**
- That understanding code is more important than speed
- That tracing is a valuable skill, not busywork
- That asking themselves (not AI) is how they learn

## Exercise Difficulty Progression

### Exercise Set 1 (Tracing) - Level Progression
- 1.1: Simple if-else (Level 1) ✓ Warm-up
- 1.2: Nested if-else (Level 2) ↑ More complex conditions
- 1.3: Simple loop (Level 2) ↑ Introduction to loops
- 1.4: Loop with conditional (Level 3) ↑ Combining structures
- 1.5: Nested loops (Level 4) ↑ Challenge/synthesis

### Exercise Set 2 (Errors) - Variety
- 2.1: Syntax-like error (semicolon)
- 2.2: Logic error (correct syntax, wrong logic)
- 2.3: Off-by-one error (classic mistake)
- 2.4: Uninitialized variable (best practice issue)

### Exercise Set 3 (Prediction) - Increasing Complexity
- 3.1: Simple conditional
- 3.2: Countdown loop
- 3.3: Loop with accumulation
- 3.4: Factorial calculation (pattern recognition)
- 3.5: Nested loop multiplication table (pattern visualization)

- Introduction & setup: 5 min

- Guided practice (1.2-1.3): 15 min
- Independent work (1.4-1.5): 15 min

- Exercise Set 1: 30-40 minutes

- Session 1: Exercise Set 1 (in-class)

## Customization Ideas

### Make It More Challenging:
- Vary the starting values in examples
- Add more complex conditions (multiple && or ||)
- Include unsigned integers or floating-point numbers
- Create your own exercises using same format

### Make It More Accessible:
- Provide partially completed traces (fill-in-the-blank)
- Give code segments to trace in smaller chunks
- Pair students for collaborative tracing
- Have students explain aloud before writing

### Make It More Relevant:
- Use context students care about (games, apps, social media)
- Modify code examples to match student interests
- Have students write trace-friendly code for each other

## Notes for Success

1. **The traces matter more than the answers**
   - A student with wrong answer but clear thinking ≠ failure
   - A student with right answer but no work = potential plagiarism

2. **Normalize mistakes**
   - Getting traces "wrong" is learning, not failure
   - Use mistakes as teaching moments
   - Celebrate effort and clarity

3. **This sets the tone**
   - How you teach Week 1 defines class culture
   - If you insist on understanding, students will provide it
   - If you accept shortcuts, shortcuts spread

4. **Consistency matters**
   - Keep the "no internet, no AI" boundary firm
   - Every exercise should require explanation
   - Recognize and reward clear thinking
