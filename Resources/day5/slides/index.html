What are Functions?

Definition: Reusable blocks of code that perform specific tasks

Benefits:

  • Code Reusability: Write once, use many times
  • Readability: Give code meaningful names
  • Maintainability: Easier to fix and update
  • Modularity: Break problems into smaller pieces

Three Types of Functions

Type 1: Void Functions
Perform a task
No return value
Example: printLine()
Type 2: Value-Returning
Compute a result
Return the result
Example: square(5)
Type 3: Parameters
Accept input data
Customize behavior
Example: add(3, 5)

Void Functions: Basic Concept

Functions that perform a task but return NO value

 1 | void printLine() {
 2 |     printf("-----\n");
 3 | }
 4 |
 5 | int main() {
 6 |     printLine();  // Call
 7 |     printLine();  // Call again
 8 |
 9 |     return 0;
10 | }
Key Points:
  • Return type: void
  • No return statement needed
  • Used for "actions" (printing, modifying)
  • Called with functionName()
Output:
-----
-----

Value-Returning Functions

Functions that compute and return a result

 1 | int square(int num) {
 2 |     int result = num * num;
 3 |     return result;
 4 | }
 5 |
 6 | int main() {
 7 |     int answer = square(5);
 8 |     // answer = 25
 9 |
10 |     printf("%d\n", answer);
11 |     return 0;
12 | }
Key Points:
  • Return type: int (or other)
  • Must have return value;
  • Value replaces function call
  • Used for "calculations"
Output:
25

Tracing Value-Returning Functions

How return values replace function calls

 1 | int add(int a, int b) {
 2 |     return a + b;
 3 | }
 4 |
 5 | int main() {
 6 |     int result = add(3, 5);
 7 |     // becomes: result = 8;
 8 |
 9 |     printf("%d\n", result);
10 |     return 0;
11 | }
Step Code result
1 int result = ? (uninitialized)
2 Call add(3, 5) ? (waiting)
3 return 3 + 5 ? (computing)
4 = 8 8 ✓
5 printf("%d", 8) 8
Output:
8

Parameters vs Arguments

Parameters: Variables in the function definition (like placeholders) 
int add(int a, int b) { // a, b are PARAMETERS
    return a + b;
}
Arguments: Actual values passed to the function when called 
int result = add(3, 5); // 3, 5 are ARGUMENTS

Pass-by-Value Semantics

How values are passed into functions

 1 | void modifyValue(int x) {
 2 |     x = 100;  // Changes copy
 3 |     printf("In fn: %d\n", x);
 4 | }
 5 |
 6 | int main() {
 7 |     int num = 5;
 8 |     modifyValue(num);
 9 |     // num is STILL 5!
10 |     printf("In main: %d\n", num);
11 |     return 0;
12 | }
Important: Parameters are copies of arguments
Location Variable Value
main() num 5
modifyValue() x (copy) 100
main() num 5 (unchanged)
Output:
In fn: 100
In main: 5

Multiple Parameters

Functions with more than one parameter

 1 | double average(double a,
 2 |                 double b) {
 3 |     return (a + b) / 2.0;
 4 | }
 5 |
 6 | int main() {
 7 |     double avg =
 8 |         average(10.0, 20.0);
 9 |     // avg = 15.0
10 |
11 |     printf("%.1f\n", avg);
12 |     return 0;
13 | }
Function Param Value
average() a 10.0
average() b 20.0
average() return 15.0
main() avg 15.0
Parameter order matters! average(10, 20) ≠ average(20, 10)
Output:
15.0

Exercise Set 1.1 Walkthrough

Common bug: Missing function call parentheses

 1 | void greet() {
 2 |     printf("Hello!\n");
 3 | }
 4 |
 5 | int main() {
 6 |     greet;     // BUG: Missing ()
 7 |     // Should be: greet();
 8 |
 9 |     return 0;
10 | }
Bug: greet (without parentheses) is just a reference to the function, NOT a call
Fix: greet() (with parentheses) executes the function
Key Lesson: Always use () to call a function!
Actual Output: (none)
Expected: Hello!

Exercise Set 2.1 Walkthrough

Common bug: Missing return statement

 1 | int multiply(int a, int b) {
 2 |     int product = a * b;
 3 |     // BUG: Missing return!
 4 | }
 5 |
 6 | int main() {
 7 |     int result = multiply(3, 4);

Bug: Function doesn't return the product (unpredictable value returned)

return product;

Exercise Set 3.2 Walkthrough

Common bug: Type mismatch in return value

 1 | int divide(int a, int b) {
 2 |     int result = a / b;
 3 |     return result;
 4 | }
 5 |
 6 | int main() {
 7 |     int ans = divide(5, 2);
 8 |     // ans = 2 (not 2.5!)
 9 |     printf("%d\n", ans);
10 |     return 0;
11 | }
Bug: Integer division truncates 5/2 to 2 (loses decimal)
Fix: Use double for decimal result 
double divide(int a, int b) {
    return (double)a / b;
}
Actual Output: 2
Expected: 2.5

Common Function Mistakes

Mistake 1: Forgetting void for functions with no return 
 1 | // WRONG:
 2 | int printMsg() {  // Says int but
 3 |     printf("Hi");
 4 |     // No return!
 5 | }
 6 |
 7 | // RIGHT:
 8 | void printMsg() {
 9 |     printf("Hi");
10 | }

Common Function Mistakes (continued)

Mistake 2: Returning from void function 
 1 | // WRONG:
 2 | void greet(char name[]) {
 3 |     printf("Hello %s\n", name);
 4 |     return name;
 5 | }
 6 |
 7 | // RIGHT:
 8 | void greet(char name[]) {
 9 |     printf("Hello %s\n", name);
10 | }

Common Function Mistakes (continued)

Mistake 3: Wrong number or type of arguments 
 1 | int add(int a, int b)
 2 |     { return a + b; }
 3 |
 4 | // WRONG CALLS:
 5 | int r1 = add(5);
 6 | int r2 = add(5, 10, 15);
 7 | int r3 = add(5, "10");
 8 |
 9 | // RIGHT CALL:
10 | int r = add(5, 10);

Understanding Function Signatures

 1 | int add(int a, int b) {
 2 | ↑   ↑      ↑  ↑
 3 | │   │      │  └─ Parameters
 4 | │   │      └───── Types
 5 | │   └──────────── Name
 6 | └──────────────── Return type
 7 |
 8 | int result = add(3, 5);
 9 |              ↑  ↑
10 |              └─ Arguments
⚠️ Return type must match what the function returns!

Function Design Checklist

  1. Choose return type: void or int/double
  2. Pick function name: Descriptive name
  3. Decide parameters: What data needed?
  4. Write the body: Implement the task
  5. Add return: If not void, return value
  6. Test function: Call with different arguments
  7. Verify behavior: Check output matches expected

Function Types Summary

Type Return Type Has Parameters Example Usage
Void (no params) void No void printLine() printLine()
Void (with params) void Yes void greet(char n[]) greet("Alice")
Returning (no params) int/double/etc No int getRandom() int x = getRandom()
Returning (with params) int/double/etc Yes int add(int a, b) int s = add(3, 5)

Key Takeaways

✓ Always use () to call functions
greet() calls the function; greet just references it
✓ Match return type to what you return
If function is int, must return an int value
✓ Parameters are copies (pass-by-value)
Changes inside function don't affect original variables
✓ Trace through function execution carefully
Follow parameter values and return values step-by-step
