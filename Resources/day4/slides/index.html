The Debugging Process

1. READ Understand what the code SHOULD do
2. PREDICT What will it ACTUALLY do?
3. TRACE Follow variables step-by-step
4. FIND Which line is buggy?
5. FIX Write the correct version
6. VERIFY Trace the fixed code

ALWAYS TRACE BEFORE GUESSING!

Exercise Set 1: Logic Operator Bugs

Category: || vs && confusion
Difficulty: Medium
Count: 2 exercises

These exercises focus on:

  • Using correct logical operators
  • Understanding range checking
  • AND (&& ) for multiple conditions
  • OR (||) for alternative conditions

Exercise 1.1: Range Check Bug

Goal: Count numbers between 5 and 15 (inclusive)

Expected: 11 numbers (5, 6, 7, ..., 15)

Actual: 19 numbers

for (int i = 1; i <= 20; i++) {
    if (i > 5 || i < 15) {
        count++;
    }
}

Exercise 1.1: Finding the Bug

Step 1: Trace a few values

i i > 5 i < 15 i > 5 || i < 15 Counted?
3 FALSE TRUE TRUE Yes (WRONG!)
5 FALSE TRUE TRUE Yes (correct)
10 TRUE TRUE TRUE Yes (correct)
16 TRUE FALSE TRUE Yes (WRONG!)

Problem: OR means "if EITHER condition is true". We need AND!

Exercise 1.1: The Fix

Buggy Code: 
if (i > 5 || i < 15) {
Fixed Code: 
if (i >= 5 && i <= 15) {

Why this works:

  • Both conditions must be true (AND)
  • Only numbers 5-15 satisfy BOTH i >= 5 AND i <= 15
  • Numbers like 3 fail the first condition
  • Numbers like 16 fail the second condition

Exercise Set 2: Counter & Accumulator Bugs

Category: Wrong variables, double operations
Difficulty: Medium
Count: 2 exercises

Common mistakes:

  • Incrementing counter twice
  • Using wrong variable for accumulation
  • Forgetting to initialize variables

Exercise 2.1: Double Counting Bug

Goal: Count numbers divisible by 3 (1-20)

Expected: 6 numbers (3, 6, 9, 12, 15, 18)

Actual: 12

if (i % 3 == 0) {
    count++;
    count++;     // ← BUG: Increments twice!
}

The Fix: Remove one of the count++; statements

Exercise 2.2: Wrong Accumulator

Goal: Sum multiples of 5 (1-50)

Expected: 275 (5+10+15+...+50)

Actual: Works, but uses WRONG variable

int sum = 0;
int count = 0;
if (i % 5 == 0) {
    count = count + i;  // ← Should be sum!
}

Subtle Bug: Output is correct by accident, but sum is never updated!

Exercise Set 3: Loop Control Bugs

Category: Iteration issues
Difficulty: Medium-Hard
Count: 2 exercises

Focus areas:

  • Wrong increment values
  • Nested loop off-by-one errors
  • Loop bounds not matching intent

Exercise 3.1: Wrong Increment

Goal: Print numbers 1 to 5

Expected: 1 2 3 4 5

Actual: 1 3 5

while (i <= 5) {
    printf("%d ", i);
    i = i + 2;    // ← BUG: Skips even numbers!
}

The Fix: Change to i = i + 1; or i++;

Exercise 3.2: Nested Loop Off-by-One

Goal: Print 3x3 grid of asterisks

Expected:

* * *
* * *
* * *

Actual: 3 rows, 2 columns

for (int col = 0; col < 2; col++) {
    printf("* ");
}

The Fix: Change to col < 3

Exercise Set 4: Boundary Condition Bugs

Category: Operator direction, edge cases
Difficulty: Hard
Count: 2 exercises

These test understanding of:

  • < vs <= (does 10 count?)
  • > vs >= (edge case handling)
  • Sentinel values

Exercise 4.1: Reversed Operator

Goal: Count numbers <= 10

Expected: 10 (numbers 1-10)

Actual: 11 (numbers 10-20)

if (i >= 10) {  // ← BUG: > instead of <
    count++;
}

Key Lesson: Pay attention to < vs > vs <= vs >=

Exercise Set 5: Complex Logic Bugs

Category: Multi-condition confusion
Difficulty: Hard
Count: 1 exercise (comparison)

Understanding:

  • AND (&& ) = Both must be true (restrictive)
  • OR (||) = Either can be true (permissive)
  • Truth tables for complex conditions

Exercise 5.1: AND vs OR

Find: Numbers that are (even AND > 10)

n Even > 10 AND OR
8 YES NO NO YES
11 NO YES NO YES
12 YES YES YES YES
5 NO NO NO NO

AND: Only 12, 14, 16, 18, 20 match (5 numbers)

OR: 2, 4, 6, 8, 10, 11, 12, 13... (15 numbers)

Exercise Set 6: Variable State Bugs

Category: Initialization, scope
Difficulty: Medium
Count: 1 exercise

Common issues:

  • Variables used without initialization
  • Forgetting to reset variables between uses
  • Wrong variable scope

Exercise 6.1: Forgotten Reset

Goal: Sum 3 separate lists independently

// List 1: sum = 6
sum = 0;
for (int i = 1; i <= 3; i++) sum += i;
printf("List 1: %d\n", sum);

// List 2: sum should be 15, but...
// Missing: sum = 0;
for (int i = 4; i <= 6; i++) sum += i;
printf("List 2: %d\n", sum);  // Prints 21!

Problem: sum still has value 6 from List 1

Result: 6 + 4 + 5 + 6 = 21 instead of 15

Tips for Finding Bugs

DO: Trace with actual values
DO: Check boundary cases (first, last, edge values)
DO: Verify your fix works
DON'T: Just look at output and guess
DON'T: Skip the tracing step
DON'T: Modify code randomly hoping it works
