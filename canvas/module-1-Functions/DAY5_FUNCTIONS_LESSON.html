What Are Functions?

Definition:

Functions are reusable blocks of code that perform specific tasks. They help organize code, improve readability, and reduce repetition.

Why Use Functions?

Benefit Explanation
Reusability Write code once, use it many times
Readability Code is easier to understand with meaningful function names
Maintainability Easier to fix bugs and make updates
Modularity Break large problems into smaller, manageable pieces

Three Types of Functions

Type 1: Void Functions

Purpose: Perform a task but return NO value

Syntax: void functionName(parameters) { ... }

Used for: Printing output, displaying messages, input operations

Example:

void printLine() {
    printf("-----\n");
}

int main() {
    printLine();  // Call the function
    return 0;
}

Type 2: Value-Returning Functions

Purpose: Perform a calculation and return a result

Syntax: int functionName(parameters) { return value; }

Used for: Calculations, conversions, computations

Example:

int square(int num) {
    return num * num;
}

int main() {
    int result = square(5);  // result = 25
    printf("%d\n", result);
    return 0;
}

Type 3: Functions with Parameters

Purpose: Accept input data and use it in the function

Syntax: void greet(char name[]) { ... }

Parameters: Variables in function definition (placeholders)

Arguments: Actual values passed when calling the function

Example:

void greet(char name[]) {      // Parameters
    printf("Hello, %s!\n", name);
}

int main() {
    greet("Alice");  // Arguments: "Alice"
    greet("Bob");    // Arguments: "Bob"
    return 0;
}

Parameters vs Arguments

Parameters

Variables in the function definition

Like placeholders or templates

void add(int a, int b) {
    // a, b are PARAMETERS
}

Arguments

Actual values passed to the function when called

Real data that gets used

int result = add(3, 5);
// 3, 5 are ARGUMENTS

Pass-by-Value Semantics

Key Concept:

When you pass a variable to a function, the function receives a COPY of the value, not the original variable.

Example:

 1 | void increment(int x) {
 2 |     x = x + 1;              // Changes the COPY only
 3 |     printf("Inside: %d\n", x);
 4 | }
 5 |
 6 | int main() {
 7 |     int num = 5;
 8 |     printf("Before: %d\n", num);
 9 |
10 |     increment(num);          // Pass copy of num
11 |
12 |     printf("After: %d\n", num);  // num is STILL 5!
13 |     return 0;
14 | }

Output:

Before: 5
Inside: 6
After: 5

Important: Changes inside the function do NOT affect the original variable in main()

│   │      └───── Parameter types: what kind of data

Parts Explained:

Part Example Meaning

Function Name add Descriptive name of what it does
Parameters int a, int b Two integers needed for the function

Common Function Mistakes

Mistake 1: Missing () When Calling Functions

WRONG:

greet;  // No ()

RIGHT:

greet();  // With ()

Mistake 2: Missing Return Statement

WRONG:

int add(int a, int b) {
    int sum = a + b;
    // Missing return!
}

RIGHT:

int add(int a, int b) {
    int sum = a + b;
    return sum;
}

Mistake 3: Wrong Number of Arguments

WRONG:

int result = add(5);  // Missing one!
int r2 = add(1, 2, 3);  // Too many!

RIGHT:

int result = add(3, 5);  // Correct
int r2 = add(1, 2);      // Correct

Tracing Function Execution

Let's trace through this program step-by-step:

 1 | int add(int a, int b) {
 2 |     return a + b;
 3 | }
 4 |
 5 | int main() {
 6 |     int result = add(3, 5);
 7 |     printf("%d\n", result);
 8 |     return 0;
 9 | }

Step-by-Step Execution:

Step What Happens Variables
1 Enter main() (none yet)
2 Call add(3, 5) - enter function a=3, b=5
3 Calculate a + b (3 + 5 = 8) a=3, b=5, result=?
4 Return 8 from add(), exit function result gets 8
5 Print result (8) result=8
6 Return 0, exit main() Program ends

Output:

8

Key Takeaways

✓ Functions organize code into reusable pieces

Instead of writing the same code multiple times, write a function and call it

✓ Use void for actions, return types for calculations

If function just does something (print, input), use void. If it computes a value, return it.

✓ Parameters let functions accept different data

Parameters are placeholders that get filled with actual values when the function is called

✓ Always use () to call functions

functionName; is a reference. functionName(); is a call. You need the parentheses!

✓ Parameters are copies (pass-by-value)

Changes to parameters inside a function do NOT affect the original variables
