Relational Operators

Definition:

Relational operators compare two values and return a boolean result (true or false, represented as 1 or 0 in C).

Six Basic Relational Operators

Operator Name Example Result (if a=5, b=3)
== Equal to a == b false (0)
!= Not equal to a != b true (1)
< Less than a < b false (0)
<= Less than or equal a <= b false (0)
> Greater than a > b true (1)
>= Greater than or equal a >= b true (1)

Real-World Example

#include <stdio.h>

int main() {
    int age = 16;

    if (age >= 18) {
        printf("You can vote\n");
    } else {
        printf("You cannot vote yet\n");
    }

    return 0;
}

Logical Operators

Definition:

Logical operators combine multiple conditions to create more complex boolean expressions.

The Three Logical Operators

1. AND Operator (&&)

Purpose: Returns true ONLY if BOTH conditions are true

Syntax: condition1 && condition2

Truth Table:

Condition 1 Condition 2 Result
T T T
T F F
F T F
F F F

Example:

int age = 20;
int hasLicense = 1;

if (age >= 18 && hasLicense == 1) {
    printf("You can drive\n");  // TRUE - both conditions met
}

2. OR Operator (||)

Purpose: Returns true if AT LEAST ONE condition is true

Syntax: condition1 || condition2

Truth Table:

Condition 1 Condition 2 Result
T T T
T F T
F T T
F F F

Example:

int isWeekend = 1;
int isHoliday = 0;

if (isWeekend || isHoliday) {
    printf("No work today!\n");  // TRUE - at least one is true
}

3. NOT Operator (!)

Purpose: Reverses/negates a boolean value

Syntax: !condition

Truth Table:

Condition NOT Condition
T F
F T

Example:

int isRaining = 0;

if (!isRaining) {
    printf("Let's play outside!\n");  // TRUE - it's NOT raining
}

Operator Precedence & Hierarchy

Key Concept:

Operators have an order of evaluation. Knowing precedence helps you predict what complex expressions will do.

Order of Operations (Highest to Lowest Priority)

Priority Operators Description
1 (Highest) ! NOT operator (evaluated first)
2 *, /, % Multiplication, division, modulo
3 +, - Addition, subtraction
4 <, <=, >, >= Relational operators (comparison)
5 ==, != Equality operators
6 && AND operator (evaluated before OR)
7 (Lowest) || OR operator (evaluated last)

Precedence Example

int a = 5, b = 10, c = 15;

// Evaluate: a < b && b < c || a > c
// Step 1: Evaluate relational operators first
//   a < b = true (5 < 10)
//   b < c = true (10 < 15)
//   a > c = false (5 > 15)
// Step 2: Evaluate && (AND has higher precedence than OR)
//   true && true = true
// Step 3: Evaluate || (OR)
//   true || false = true
// RESULT: true

Use Parentheses for Clarity

// Hard to read - relies on precedence knowledge
if (a > 5 && b < 20 || c == 0) { }

// Easy to read - explicitly shows intent
if ((a > 5 && b < 20) || (c == 0)) { }

Real-World Examples: Multiple Operations

Example 1: Age Validation

int age = 22;

// Check if age is valid (between 0 and 150)
if (age > 0 && age < 150) {
    printf("Valid age\n");
} else {
    printf("Invalid age\n");
}

Example 2: Grade Assignment

int score = 85;

// Determine grade: A if 90+, B if 80-89, C if 70-79, etc.
if (score >= 90) {
    printf("Grade: A\n");
} else if (score >= 80 && score < 90) {
    printf("Grade: B\n");
} else if (score >= 70 && score < 80) {
    printf("Grade: C\n");
} else {
    printf("Grade: F\n");
}

Example 3: Login Validation

char username[20] = "john";
char password[20] = "secret123";
int isAdmin = 0;

// User can log in if they have correct credentials AND are not banned
// OR if they are an admin
if ((username[0] == 'j' && password[0] == 's') || isAdmin == 1) {
    printf("Login successful\n");
} else {
    printf("Login failed\n");
}

Common Mistakes

Mistake 1: Using = Instead of ==

WRONG:

if (age = 18) {  // = assigns, doesn't compare!
    printf("You are 18\n");
}

RIGHT:

if (age == 18) {  // == compares values
    printf("You are 18\n");
}

Mistake 2: Forgetting AND When Checking Ranges

WRONG:

if (score > 70 < 100) {  // Incorrect syntax
    printf("In range\n");
}

RIGHT:

if (score > 70 && score < 100) {  // Use &&
    printf("In range\n");
}

Mistake 3: Using AND When OR Is Needed

WRONG:

// "Give me a book OR a movie"
// BUT using AND means BOTH needed
if (hasBook && hasMovie) {
    printf("Entertainment available\n");
}

RIGHT:

// Either a book OR a movie (or both)
if (hasBook || hasMovie) {
    printf("Entertainment available\n");
}

Key Takeaways

✓ Relational operators compare values (==, !=, <, >, <=, >=)

They return true (1) or false (0) based on the comparison

✓ AND (&&) requires BOTH conditions to be true

Use && to check multiple conditions that must all be satisfied

✓ OR (||) requires AT LEAST ONE condition to be true

Use || when any of the conditions being satisfied is enough

✓ NOT (!) reverses boolean values

Use ! to check when something is NOT true

✓ Remember operator precedence

NOT → comparison → AND → OR. Use parentheses for clarity!
